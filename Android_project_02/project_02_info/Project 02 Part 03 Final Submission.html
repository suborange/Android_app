Project 02: Part 03 Final Submission

  • Due Apr 22 by 11:59pm
  • Points 200
  • Submitting a file upload
  • Available until Apr 22 at 11:59pm

We are in the endgame now. Note: I will NOT answer questions about this project the day it is due.

There are several required items and several optional items.  If any of the required items are missing the assignment will be marked as 0.  You may choose whichever of the optional items you'd like to include. The maximum possible grade for this assignment is 220 (110%).  To earn points for any of the items on the rubric they must be CLEARLY demonstrated in the required video.  I'd suggest uploading the video to Youtube and using timestamps. 

 

Goal and Learning Outcomes

Show that you have learned how to write Java and develop an Android application. Also demonstrate the creation and application of use cases. In short, this project encompases everything this course is intended to cover.

 

Learning outcomes addressed

 

CLO1: Object Oriented Software Design

A.    Application of Inheritance and Interfaces

B.   Analysis and application of abstraction

C.   Comprehend and discuss software life cycles

D.   Comprehend and discuss different software development process models

E.    Apply use case modeling techniques.

F.    Develop a software system using design patterns. 

CLO2 Knowledge of the Java Language

A.    Comprehend, Develop, and Evaluate a Java program

B.   Apply predefined classes in a Java project.

C.   Develop, use, and analyze custom java classes.

D.   Apply Java programming skills to programming projects.

E.    Use Inner Classes in programming assignments

F.    Define, explain and apply Java Collections.

 CLO3 Knowledge of the Android Platform

A.   Develop and appraise an Android app using Android Studio.

B.   Apply object-oriented design to Android development

 

What we will do

 

Finish our Android Shopping app and turn it in. 

Grades

  1. Do the use cases pass?
  2. Are the use cases descriptive, rigorous, and show unique functionality? 
    1. See below for an example of a rigorous use case
    2. e.g. creating a user is a fine use case so long as creation is shown, the app is force quit, and the user is showed to still exists. A derivative use case would be to create an entirely NEW use case to show creating a DIFFERENT user.
  3. I want a PDF of the document . Any other document will result in a loss of points.

 

What you submit

  • 1 (one) pdf document with a github link, ERD diagrams, and lists all of the use cases
    • It should be your copy of this document
    • Download the completed form as a pdf and submit it with your assignment.
  • 1 .zip file containing the android application
    • It MIGHT be necessary to include a link to a google drive folder with these files. If I can't access the files you will get a 0 for this assignment.
  • A link to the video
    • Either upload it to Youtube or put it in a google drive folder

 

Rigorous Use Case Example

Create a new user 

  1. Attempt to login with username
    1. fail to show username isn't taken
  2. Click 'Create new account'
  3. Enter username
    1. optional fail attempt: enter a username that exists
  4. Enter password
    1. optional fail attempt: show password that doesn't match criteria
  5. click 'Create'
  6. return to login page
  7. force quit app
  8. login with new user
